MCQ & Option Shuffling Mechanics

Every test attempt leverages an advanced Fisher-Yates Shuffling protocol running client-side. The sequence of clinical questions is completely randomized on every attempt, providing a fresh challenge and preventing visual key memorization.

// Shuffling integrity update flow: 1. Questions loaded from JSON source files are scrambled into random array index positions.
2. Options (A, B, C, D) are shuffled dynamically per question.
3. The system maps the original correct answer's string to its new position.
4. Correct answer index is recalculated dynamically prior to score analysis.

This ensures that options like "All of the above" or "A and B only" are converted to accurate specific statements and shuffled flawlessly while preserving absolute scoring accuracy down to the millisecond.

Offline-First Capabilities & Storage Security

We prioritize study continuity. If you lose connection or haven't connected Firebase Firestore credentials, the application switches seamlessly to Offline Mode. All attempts, study timer records, bookmarks, and completions are stored immediately inside your browser's Local Storage sandbox, preserving your hard work until you decide to connect a cloud backend.
